# Phase 7 Plan 01: Chart Lifecycle Persistence and Incremental Render

## Objective

- **What:** Refactor chart/grid rendering so chart instances preserve runtime state during tag additions and frequency changes.
- **Why:** Current lifecycle destroys cards, drops in-memory series, and forces visible reload regressions.
- **Output:** Stable chart-card identity and non-destructive update path for tag/frequency mutations.
