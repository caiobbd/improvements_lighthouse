# Phase 06.2.1 Context — Sidebar continuity + non-disruptive chart loading

## Why this patch now

Current UX has two high-impact continuity breaks for operational users:

1. Sidebar scroll resets to top after interactions (expand/select/plot/drag), forcing repeated navigation through deep trees.
2. Chart loading replaces the chart body with message/blank state, hiding axis context and previously loaded data during fetches.

These regressions reduce trust and slow down decision workflows for users working with many assets/sensors.

## Requested outcomes

- Equipment and Sensors/Events sidebars must preserve scroll position during all local interactions and rerenders.
- Charts must always render a visible canvas with X-axis context (page range or active zoom domain), including when series is empty.
- Fetching new data should be non-destructive:
  - Existing series remain visible while loading.
  - A loading indicator is shown above/over chart area without replacing current plot.
  - Empty chart flow remains continuous: empty graph -> loading indicator -> plotted series.

## Technical scope

- Frontend-only patch (no backend contract changes required).
- Main files expected in scope:
  - `frontend/charts/app.js`
  - `frontend/charts/components/chart-card.js`
  - `frontend/charts/components/d3-line-chart.js`
  - `frontend/charts/styles/components.css`

## Risks / impacts to control

- Scroll persistence must not break current virtual rerender behavior or produce stale references.
- Non-destructive chart loading must preserve stale-request guards and avoid race-condition flashes.
- Drag/drop and right-click plotting flows must remain unchanged functionally.
- Interaction domains (zoom/pan/sync) must remain stable when loading overlay is visible.

## UAT focus

- Deep-tree navigation continuity after repeated expand/select/plot/drag actions.
- Sensor/events sidebar continuity after switching tabs and plotting.
- Empty-chart and existing-chart loading transitions with visual continuity.
- No regressions in drag-drop, tag add/remove, and sync zoom behavior.
