# Phase 6 Plan 01: Backend Intel Events Contracts

## Objective

- **What:** Add backend support for Shape Intel event ingestion, normalization, subtree-aware filtering, and event payload contracts.
- **Why:** Frontend event tab and alarm-page generation need a stable, fast contract that does not trigger repeated expensive retrieval per interaction.
- **Output:** New events APIs for sidebar consumption and alarm plotting inputs.
