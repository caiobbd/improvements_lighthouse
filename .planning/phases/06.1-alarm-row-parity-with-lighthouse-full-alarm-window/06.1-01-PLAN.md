# Phase 6.1 Plan 01: Alarm Display Contract and Formatting Helpers

## Objective

- **What:** Define and centralize the alarm display contract used by the alarm information row.
- **Why:** Lighthouse parity requires a predictable hierarchy and consistent fallback behavior even when payload fields are sparse.
- **Output:** Shared helper path that prepares alarm metadata, warnings, labels, and timing values before UI rendering.
