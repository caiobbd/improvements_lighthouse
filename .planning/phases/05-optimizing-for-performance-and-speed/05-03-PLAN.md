# Phase 5 Plan 03: Batched Fetch and Refresh UX

## Objective

- **What:** Implement per-chart timeseries batching and progressive refresh UX without blanking existing charts.
- **Why:** N-per-tag fetch fanout and hard loading states were increasing latency and reducing perceived responsiveness.
- **Output:** New `/timeseries-batch` API, frontend batch client usage, stale-request guard, and global refresh progress bar.
